# Roadmap

## Month 1: open-source readiness

- Keep Apache-2.0 licensing and branding policy clear.
- Finish README, install, privacy, security, and trust-model documentation.
- Add release checklist and basic issue templates.
- Keep generated binaries and private config out of the repository.

## Month 2: beta onboarding

- Improve sync install diagnostics.
- Add screenshots and a short user guide.
- Improve `/mpf status` and sync status output.
- Collect beta feedback from real Mythic+ runs.

## Month 3: validation layer

- Improve server-side run validation.
- Add clearer rejection reasons.
- Add tests for combat-log parsing and snapshot generation.

## Month 4: public sync mode

- Document the public HTTPS API mode.
- Keep local tunnel support only as a development option.
- Add release checksums and verification instructions.

## Month 5: contributor experience

- Add development setup docs.
- Add parser fixtures and regression tests.
- Improve issue templates and triage labels.

## Month 6: season-ready release

- Update compatibility for the current Retail build.
- Prepare clean release notes.
- Review privacy, security, and packaging before public promotion.
